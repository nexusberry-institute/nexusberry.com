Part 2

HTML Document Structure

Every HTML page follows the same blueprint — learn it once, use it forever

The HTML Blueprint

<!DOCTYPE html> + <html lang="en">
<head>
🎬 Backstage (metadata)
<body>
🎭 The Stage (visible)
💡 <head> = browser/SEO info. <body> = what users see.

What Each Part Does

Element Purpose
<!DOCTYPE html> Tells browser: "This is HTML5"
<html lang="en"> Root element + language for accessibility
<meta charset="UTF-8"> Support for all characters (é, ñ, etc.)
<meta name="viewport"...> Makes page mobile-responsive
<title> Text shown in browser tab

Let's build our first HTML page!

→ Switch to VS Code

Spot the Error


<html>
<head>
    <title>My Page<title>
</head>
<body>
    <h1>Welcome</h1>
</body>
</html>

What's wrong with this code? (Two issues!)

  1. Missing <!DOCTYPE html> only
  2. The <title> closing tag is wrong only
  3. Both: missing <!DOCTYPE html> AND <title> should be </title>
  4. Nothing is wrong — this is valid HTML

Part 2 Summary

  • Common Mistake: Forgetting <!DOCTYPE html> — causes quirks mode
  • Best Practice: Always include lang, charset, and viewport meta
  • Pro Tip: Use Emmet (! + Tab) AFTER you understand the structure — never before

Part 3

Common HTML Elements

The building blocks of every webpage

~20 elements cover most builds 90%
Total HTML elements (rarely used) 100+
💡 Focus on the core 20. That's what pros use daily.

Headings: h1 through h6


<h1>Main Page Title</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Detail Heading</h4>

One <h1> per page. Never skip levels.

Paragraphs, Links & Images


<p>This is a paragraph of text.</p>

<a href="https://nexusberry.com">Visit NexusBerry</a>

<img src="https://picsum.photos/600/400"
     alt="A beautiful landscape">

Links need https://. Images need alt text. Always.

Lists: Ordered & Unordered


<!-- Bullets -->
<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>
</ul>
                    

<!-- Numbered -->
<ol>
    <li>Learn HTML</li>
    <li>Learn CSS</li>
    <li>Learn JS</li>
</ol>

Let's code these elements!

→ Switch to VS Code

Spot the Error


<a href="nexusberry.com">Visit NexusBerry</a>

This link won't work as expected. Why?

  1. The tag name should be <link> not <a>
  2. Missing https:// — browser treats it as a relative path
  3. Missing target="_blank" attribute
  4. Link text should be inside a <span>

Part 3 Summary

  • Common Mistake: Skipping heading levels (h1 → h3) — bad for SEO and accessibility
  • Best Practice: Always add alt text to images — it's not optional
  • Pro Tip: "I've seen companies get legal notices for missing alt text. Make it a habit from day one."

Building the Blog Article

↓

Let's build the Blog Article!

→ Switch to VS Code — live coding

<strong> vs <b>

We used <strong> for the author name. What's the difference?

<strong>

Semantic: "important"

Screen readers + SEO understand it

VS

<b>

Visual only: "bold"

No meaning to assistive tech

✅ Use <strong> when the text is important. Both look bold, but only <strong> has meaning.

Part 4 Summary

  • Common Mistake: Using <br> for spacing instead of separate <p> tags
  • Best Practice: One <h1> per page, proper heading hierarchy throughout
  • Pro Tip: "The blog article you just built follows the same HTML structure as the New York Times. The difference is CSS — which comes in Lecture 3."

Cheat Sheet

HTML Document Skeleton


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- content here -->
</body>
</html>

Common Elements Reference

Element Purpose
<h1>-<h6> Headings (h1 = most important)
<p> Paragraph
<a href=""> Hyperlink
<img src="" alt=""> Image (self-closing)
<ul> / <ol> Unordered / Ordered list
<li> List item
<strong> Important text (bold)
<em> Emphasized text (italic)
