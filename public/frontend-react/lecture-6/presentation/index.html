Part 1

Why Tailwind?

The Utility-First Revolution

From writing CSS to composing with class names

Traditional CSS vs Tailwind

🎨 Traditional CSS

1. Think of a class name

2. Write CSS rules

3. Apply class in HTML

4. Debug across files


.btn-primary {
  padding: 0.75rem 2rem;
  background: #f59e0b;
  color: #000;
  font-weight: 700;
  border-radius: 0.5rem;
  /* ...12+ more lines */
}
VS

⚡ Tailwind CSS

1. Apply utility classes

2. Done!

 

 


<button class="bg-amber-500
  text-black font-bold
  py-3 px-8 rounded-lg">
  View Collection
</button>

/* Zero CSS written! */

Why Developers Love Tailwind

📋 No Naming Classes No more .card-wrapper-inner-content. Just apply what you need.
⚡ No Context Switching Stay in your HTML. No jumping to CSS files.
🎨 Built-in Design System Constrained palette, spacing scale, consistent look.
⚠ Not inline styles! Tailwind has responsive prefixes (md:), hover states (hover:), and design constraints.

Tailwind CSS 4 Setup (CDN)


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0">
    <title>My Tailwind Page</title>
    <script src="https://cdn.jsdelivr.net/npm/
      @tailwindcss/browser@4"></script>
</head>
<body class="bg-slate-950 text-gray-100">
    <!-- Your content -->
</body>
</html>
⚡ One script tag. No npm. No config. That's it.

🤔 Interactive Question

What's the main advantage of utility-first CSS over traditional CSS?

  1. It makes your HTML smaller
  2. No naming classes, no context-switching, built-in design system
  3. It replaces HTML entirely
  4. It only works with React

✅ Answer: B — No naming, no file-switching, constrained design system!

🔎 Spot the Error


<div class="bg-blue text-white p4">Hello</div>

This code has TWO errors. Can you spot them?

  1. bg-blue needs a shade (bg-blue-500), p4 needs a dash (p-4)
  2. text-white should be color-white
  3. There's no <div> in Tailwind
  4. You need a class="tw" prefix

✅ Answer: A — Colors need shades, spacing needs dashes!

💡 The Hidden Fact

Lecture 5's button: 15 lines of CSS


.btn-primary {
  display: inline-block;
  padding: 1rem 2rem;
  background-color: #f59e0b;
  color: #000;
  font-weight: 700;
  text-transform: uppercase;
  letter-spacing: 0.1em;
  border-radius: 0.5rem;
  border: none;
  cursor: pointer;
  transition: background-color 0.3s ease,
              transform 0.2s ease;
}
.btn-primary:hover {
  background-color: #d97706;
  transform: translateY(-2px);
}

Tailwind: bg-amber-500 text-black font-bold py-3 px-8 rounded-lg hover:bg-amber-600 hover:-translate-y-0.5 transition duration-300

Same result. Zero CSS. Every utility maps to CSS you already know.

Part 1 Summary

  • Common Mistake: Forgetting shade number (bg-blue ❌ → bg-blue-500 ✅)
  • Common Mistake: Missing dash in spacing (p4 ❌ → p-4 ✅)
  • Best Practice: Install Tailwind CSS IntelliSense in VS Code
  • Pro Tip: Every Tailwind class maps 1-to-1 to CSS you already know

Layout, Spacing & Sizing

Utilities

Flexbox, Grid, and the 4px spacing system

Tailwind Class Anatomy


p-4
Property (padding) Separator Value (1rem = 16px)

Pattern: {property}-{value} → p-4 = padding: 1rem

⚡ The 4px base: number × 4 = pixels. p-6 = 6 × 4 = 24px = 1.5rem

Spacing Scale

p-1 4px 0.25rem
p-2 8px 0.5rem
p-4 16px 1rem
p-6 24px 1.5rem
p-8 32px 2rem
p-12 48px 3rem

Same prefixes: p-, px-, py-, pt-, m-, mx-, my-, mt-, mb-

🤔 Concept Check

What does max-w-7xl mx-auto do?

  1. Makes text bold and centered
  2. Sets max-width to 80rem and centers the block — the standard container pattern
  3. Creates a flexbox container
  4. Adds maximum width only on XL screens

✅ Answer: B — Same as Lecture 5's max-width + margin: 0 auto!

Part 2 Summary

  • Container Pattern: max-w-7xl mx-auto px-6 (memorize this!)
  • Spacing Math: number × 4 = pixels (p-8 = 32px)
  • Pro Tip: inset-0 = top-0 right-0 bottom-0 left-0
  • Overlay: bg-black/60 replaces ::before from Lecture 5

State Modifier Anatomy

State Modifiers

🔎 focus: Input is selected. focus:ring-2
👉 active: Being clicked. active:scale-95
❌ Common mistake: hover-bg-blue-700 (dash) ❌ → hover:bg-blue-700 (colon) ✅

Hover Effects: Lecture 5 vs Tailwind

Lecture 5 CSS


.card {
  transition: transform 0.3s,
    box-shadow 0.3s;
}
.card:hover {
  transform: translateY(-8px);
  box-shadow: 0 12px 40px
    rgba(0,0,0,0.3);
}
VS

Tailwind


class="hover:-translate-y-2
  hover:shadow-xl
  transition duration-300"

🤔 Predict the Output


<div class="hover:bg-slate-800 hover:scale-105
            transition duration-300">
    Card content

What happens on hover?

  1. Only the background changes
  2. Only the scale changes
  3. Both: background darkens AND card scales up, smoothly over 300ms
  4. Error — can't have two hover: modifiers

✅ Answer: C — Multiple hover: modifiers work independently!

🔎 Spot the Error


<button class="bg-blue-500 hover-bg-blue-700
               text-white font-bold py-2 px-4 rounded">
    Click Me
</button>

What's wrong with this code?

  1. bg-blue-500 should be background-blue-500
  2. hover-bg-blue-700 uses a dash — should be hover:bg-blue-700
  3. rounded needs a size like rounded-lg
  4. Tailwind doesn't support hover effects

✅ Answer: B — Modifiers use COLON, not dash!

⚡ Quick-Fire: CSS → Tailwind

Convert Lecture 5 hover CSS to Tailwind:


.card:hover {
    transform: translateY(-8px);
    box-shadow: 0 12px 40px rgba(0,0,0,0.3);
}
.card { transition: transform 0.3s, box-shadow 0.3s; }

Answer:

hover:-translate-y-2 hover:shadow-xl transition duration-300
translateY(-8px)→-translate-y-2
box-shadow: large→shadow-xl
transition: 0.3s→transition duration-300

Part 3 Summary

  • Modifier Syntax: hover:property-value (COLON, not dash)
  • Colors: Always include shade: bg-blue-500, not bg-blue
  • Transitions: Always pair hover: transforms with transition duration-300
  • Pro Tip: bg-black/60 = black at 60% opacity (replaces rgba())

Responsive Grid: Lecture 5 vs Tailwind

Lecture 5: 3 Media Queries


.grid {
  grid-template-columns: 1fr;
}
@media (min-width: 768px) {
  .grid {
    grid-template-columns:
      repeat(2, 1fr);
  }
}
@media (min-width: 1024px) {
  .grid {
    grid-template-columns:
      repeat(3, 1fr);
  }
}
VS

Tailwind: 1 Line


class="grid
  grid-cols-1
  md:grid-cols-2
  lg:grid-cols-3
  gap-8"

Same result. Same CSS under the hood.

At a viewport of 800px, how many columns?

  1. 1 column
  2. 2 columns
  3. 3 columns

✅ Answer: B — md: (768px) applies. lg: (1024px) doesn't yet.

🤔 Concept Check

Is Tailwind's md: prefix mobile-first or desktop-first?

  1. Desktop-first — targets screens BELOW 768px
  2. Mobile-first — targets screens AT or ABOVE 768px
  3. Neither — targets exactly 768px only
  4. It depends on the class used

✅ Answer: B — Same as @media (min-width: 768px) from Lecture 5!
