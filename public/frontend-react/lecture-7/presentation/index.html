Anatomy of @theme


@theme {
    --color-brand: #8B1A1A;
}
Directive Namespace prefix Token name Color value
⚡ --color-brand auto-generates: bg-brand, text-brand, border-brand, ring-brand — the entire utility ecosystem!

How @theme Works

📝 Define in @theme --color-sage: #7C9070
→
⚙ Tailwind Generates bg-sage, text-sage, ...
→
💻 Use in HTML class="bg-sage"
→
🔄 Change Once Updates everywhere

CDN Setup with @theme


<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>
<style type="text/tailwindcss">
    @theme {
        --color-brand: #8B1A1A;
        --color-brand-light: #A52A2A;
        --color-brand-dark: #6B1010;
        --color-sage: #7C9070;
        --color-cream: #FFF8F0;
        --color-charcoal: #2C2C2C;
        --color-warm-gray: #6B6360;
    }
</style>
⚠ Must be <style type="text/tailwindcss"> — regular <style> won't process @theme!

🤔 Interactive Question


@theme {
    --color-sage: #7C9070;
}

What does this single line do?

  1. Creates only a CSS variable named --color-sage
  2. Creates a CSS variable AND generates bg-sage, text-sage, border-sage, etc.
  3. Replaces the default green color in Tailwind
  4. Only works with npm, not CDN

✅ B) Variable + ALL utility classes — one line, unlimited utilities!

🔎 Spot the Error


@theme {
    color-brand: #8B1A1A;
}

Why won't bg-brand work?

  1. The hex code is invalid
  2. Missing -- prefix — must be --color-brand
  3. @theme doesn't support colors
  4. Need to use rgb() instead of hex

✅ B) CSS custom properties ALWAYS require the -- prefix!

:root vs @theme

Lecture 5: :root

VS

Lecture 7: @theme

Creates CSS variable

Use with var() too

+ ALL Tailwind utilities!

💎 @theme is :root on steroids — it speaks Tailwind's language.

Part 1 Summary

  • Common Mistake: Forgetting type="text/tailwindcss" on the style tag

  • Pro Tip: Always add via-* for smoother gradients

Part 3

Component Extraction with @apply

Stop repeating yourself — write once, use everywhere

The Repetition Problem

❌ Without @apply

15 classes × 6 cards

= 90 class instances

Change hover? Edit 6 places.

VS

✅ With @apply

1 .menu-card × 6

= 6 clean references

Change hover? Edit 1 place.

Anatomy of @apply


.menu-card {
    @apply bg-white rounded-xl shadow-md overflow-hidden
           hover:-translate-y-1 hover:shadow-xl
           transition duration-300;
}
Class name @apply directive Utility classes Modifiers work too!

The @apply Workflow

🔍 Identify repeated pattern
↓
📝 Create named class
↓
🔨 Use @apply with utilities
↓
💻 Apply class in HTML
↓
🔄 Change once, updates everywhere

Component Examples


/* Navigation */
.nav-link {
    @apply text-warm-gray text-sm uppercase tracking-widest
           hover:text-brand transition duration-200;
}

/* Buttons */
.btn-primary {
    @apply bg-brand text-cream font-semibold py-3 px-8 rounded-lg
           hover:bg-brand-light transition duration-300;
}

/* Badges — base + variants */
.badge { @apply text-xs font-semibold px-3 py-1 rounded-full uppercase; }
.badge-spicy { @apply bg-red-100 text-red-700; }
.badge-veg { @apply bg-green-100 text-green-700; }
⚠ Rule of 3+: Only extract if a pattern repeats 3 or more times. Don't @apply everything!

Let's extract components!

Switch to VS Code — nav-link, menu-card, badges, buttons

🤔 Concept Check

Can you use hover: modifiers inside @apply?

  1. No — modifiers only work on inline utility classes
  2. Yes — ALL Tailwind modifiers work inside @apply
  3. Only hover:, not focus: or md:
  4. Only with the npm version, not CDN

✅ B) hover:, focus:, md:, lg:, dark: — all work inside @apply!

🤔 Predict the Output


.btn-primary {
    @apply bg-brand text-cream py-3 px-8 rounded-lg;
}

What HTML applies this?

  1. <button class="bg-brand text-cream py-3 px-8 rounded-lg">
  2. <button class="btn-primary">Order Now</button>
  3. <button apply="btn-primary">
  4. <button style="btn-primary">

✅ B) Just one class — all styles applied automatically!

⚡ When to @apply?

ScenarioDecision
Nav link used 4+ times✅ @apply
Unique hero section padding❌ Keep utility
Menu card used 12 times✅ @apply
One-off text color❌ Keep utility
⚡ Rule of 3+: Repeats 3 or more times? Extract. Otherwise, keep inline.

Part 3 Summary

  • Common Mistake: Over-extracting — .red-text { @apply text-red-500; } is pointless
  • Best Practice: Name by purpose: .nav-link, .menu-card — not .rounded-shadow-hover
  • Pro Tip: Base + variant pattern for families (.badge → .badge-spicy)

Overriding @apply with Inline


/* @apply default: */
.btn-primary { @apply bg-brand text-cream ...; }
                    

<!-- Inline override for dark section: -->
<button class="btn-primary bg-cream text-brand">
    Book a Table
</button>

Inline utilities always override @apply defaults — the base + context pattern.
