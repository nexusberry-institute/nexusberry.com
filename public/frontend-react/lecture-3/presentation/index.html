Part 2

CSS Selectors & The Cascade

Targeting the right elements with precision

CSS Selector Types

TypeSyntaxTargets
Elementp { }All <p> tags
Class.highlight { }Elements with class="highlight"
ID#header { }The ONE element with id="header"
Groupingh1, h2 { }Multiple selectors at once
Pseudo-classa:hover { }Elements in a specific state

Class vs ID

.class (Reusable)

Multiple elements can share it

Specificity: 0-1-0

Use for: styling

.card { color: blue; }
VS

#id (Unique)

Only ONE per page

Specificity: 1-0-0

Save for: JavaScript

#main { color: red; }
💎 Professional teams rarely use ID selectors in CSS. Classes are king.

Specificity: The 0-0-0 System


Specificity Score:   ID  -  Class  -  Element
                      0  -    0    -     0
ID = 1-0-0 Class = 0-1-0 Element = 0-0-1
🏁 p 0-0-1
→
🏆 .card 0-1-0
→
🥇 #main 1-0-0

Higher specificity ALWAYS wins. Equal? Last rule wins (the cascade).

💻 Let's see this in code!

Switch to VS Code — testing selectors & specificity

🤔 Predict the Output


p { color: blue; }
.special { color: red; }
                    

<p class="special">What color am I?</p>
  1. Blue (element selector)
  2. Red (class beats element) ✅
  3. Purple (they mix)
  4. Black (conflict = default)

Class (0-1-0) beats element (0-0-1) — specificity wins!

⚡ Quick-Fire: Rank by Specificity

Lowest to highest:


#header { }     /* ??? */
.nav-link { }   /* ??? */
p { }           /* ??? */

Answer:

p 0-0-1
→
.nav-link 0-1-0
→
#header 1-0-0

🔎 Spot the Error


<p class=".highlight">Styled text</p>

The dot . is CSS syntax, NOT HTML!


<!-- WRONG -->  <p class=".highlight">
<!-- CORRECT --> <p class="highlight">
❌ The #1 selector mistake beginners make. Dot belongs in CSS only!

💡 The Cascade Revealed


p { color: blue; }
p { color: red; }

Same selector, same specificity — what color?

Red!

Equal specificity → last rule wins. This is the "Cascade" in Cascading Style Sheets.

Part 2 Summary

  • Use Classes for styling — reusable & flexible
  • Specificity decides which rule wins (ID > class > element)
  • The Cascade — equal specificity? Last rule wins
⚡ When a rule doesn't apply, open DevTools Styles panel — strikethrough shows what's overridden.

Part 3

The Box Model

Every element is a rectangular box

The 4 Layers

MARGIN
BORDER
PADDING
CONTENT
Content (text, images) Padding (inner space) Border (the edge) Margin (outer space)
📸 Analogy: Photo (content) → Matting (padding) → Frame (border) → Wall space (margin)

content-box vs border-box

content-box (default)

Width = content only

Total = content + padding + border

300 + 40 + 10 = 350px!

VS

border-box (professional)

Width = everything included

Total = exactly what you set

Total = 300px

The professional reset:


*, *::before, *::after {
    box-sizing: border-box;
}

Width Calculation (content-box)

width: 300px; padding: 20px; border: 5px solid;

📎 Content 300px
+
↔ Padding 20 + 20
+
📗 Border 5 + 5
=
📏 Total 350px!
⚠ Padding & border are added on BOTH sides! That's why we get 350, not 325.

💻 Let's see this in code!

Switch to VS Code — exploring the Box Model in DevTools

🤔 Width Calculation


.box {
    width: 200px;
    padding: 15px;
    border: 5px solid black;
    /* default box-sizing: content-box */
}

Total width?

  1. 200px
  2. 215px
  3. 230px
  4. 240px ✅

200 + 15 + 15 + 5 + 5 = 240px (padding & border on BOTH sides)

🤔 Padding vs Margin

You want space BETWEEN two boxes. Which property?

  1. padding
  2. margin ✅
  3. border
  4. width
⚡ Padding = inside the border. Margin = outside the border (between elements).

💡 DevTools Box Model Visualizer

Open DevTools on ANY website — every element is a box!

🔵 Content Blue highlight
🟢 Padding Green highlight
🟠 Border Yellow highlight
🟠 Margin Orange highlight
💎 Try it on YouTube, Amazon, Google — everything is boxes! This is your X-ray vision.

Part 3 Summary

  • Every element is a box: content → padding → border → margin
  • Always use box-sizing: border-box (the professional reset)
  • Center horizontally: margin: 0 auto (needs a set width)
💎 Every CSS framework — Bootstrap, Tailwind, all of them — uses border-box as default.

⚠ Always check color contrast! Accessibility continues from HTML into CSS.
